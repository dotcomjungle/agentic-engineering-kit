# Client

The frontend SPA for the web project. Renders the UI, drives user input, consumes the server's REST + SSE contract, and keeps a coherent local view of server state.

Rationale and alternatives for these choices live in `guides/client/README.md`. This file is the live rulebook for working in `app/client/`.

## Stack

- **Vite + React 18+ + TypeScript (strict)** — no Next.js, no Remix.
- **Routing** — TanStack Router, file-based under `src/routes/`. `src/routeTree.gen.ts` is generated by `@tanstack/router-vite-plugin`; never edit by hand; gitignored.
- **Server state** — TanStack Query. Single `queryClient` in `main.tsx`.
- **API transport** — `fetch` via `apiFetch<T>` in `lib/api.ts`. Throws `ApiError` on non-2xx. No axios.
- **Streaming** — SSE consumed via `fetch` + `ReadableStream` in `lib/sse.ts`. No `EventSource`.
- **Cross-cutting UI state** — React Context. No Zustand/Redux/Jotai.
- **Styling** — Tailwind v4. No component library.
- **Testing** — Vitest + Testing Library + jsdom, co-located.
- **Lint/format** — ESLint flat config + `typescript-eslint` + Prettier.
- **Path alias** — `@/*` → `src/*`. The only alias.

## What this code owns

UI rendering, user interaction, routing, route-scoped data loading, the local cache of server state, SSE consumption + run-state reduction, form input, surfacing server errors.

## What this code does NOT own

- **Domain logic / validation** — server is authoritative. Type for shape, not for rules. Surface the server's errors; don't replicate its validation.
- **Persistence of domain data** — no IndexedDB, no `localStorage` for sessions/messages/etc. (Auth tokens and theme preference are fine.)
- **Harness** — see `app/server/`. The client only consumes the run stream.
- **Auth provider** — bring your own; wire it into `lib/api.ts`'s header resolver.

## Layout

```
src/
  main.tsx              # entry — QueryClient + RouterProvider
  routeTree.gen.ts      # generated; gitignored
  styles/index.css      # Tailwind + custom properties
  routes/               # file-based routes (TanStack Router)
    __root.tsx          # layout, providers, error boundary
  components/
    ui/                 # shared primitives (Button, Input, Dialog)
  lib/
    api.ts              # apiFetch<T> + ApiError + auth header resolver
    sse.ts              # SSE consumer
    run-reducer.ts      # SSE event → Run view-model (pure)
    queries.ts          # query-key factory + useXxx hooks + mutations
    types.ts            # shared types (event union, domain shapes)
  contexts/             # only for genuinely cross-cutting UI state
  hooks/                # shared React hooks
```

## Conventions

### TypeScript

- `strict: true`, `noUnusedLocals`, `noUnusedParameters`, `noFallthroughCasesInSwitch`. Type errors are build failures.
- No `any`. Use `unknown` and narrow.

### Routes (TanStack Router)

- One file per route under `src/routes/`. File naming follows TanStack Router's conventions (`sessions.tsx`, `sessions.$sessionId.tsx`, `__root.tsx`).
- A route's `loader` calls `queryClient.ensureQueryData(...)` for data the route needs; the component reads the same data with `useQuery(...)` so the cache is shared.
- Read params via the route's typed `useParams()` — never cast.

### TanStack Query

- Build query keys through the `qk` factory in `lib/queries.ts` (e.g. `qk.sessions.detail(id)`). Never inline key tuples in components.
- Mutations invalidate **specific** keys on success — never blanket `queryClient.invalidateQueries()`.
- Defaults: `staleTime` of a few seconds for lists, longer for stable details; `retry: 1` for queries; `retry: false` for mutations.
- Don't mirror server data into Context or local state. The cache is the source of truth.

### API client

- All HTTP goes through `apiFetch<T>(path, init)` in `lib/api.ts`. Never call `fetch` directly from a component or query function.
- `apiFetch` prepends the base URL, attaches auth headers from the single resolver, parses JSON, and throws `ApiError` on non-2xx.
- New API surface = a function in `lib/queries.ts` that wraps `apiFetch` and is consumed by `useQuery` / `useMutation`. Don't sprinkle `apiFetch` across components.

### SSE consumption

- All SSE goes through `consumeRun(url, { onEvent, signal, lastEventId })` in `lib/sse.ts`. Don't write a second SSE consumer.
- The event union type matches the server's event taxonomy — see `guides/harnesses/agent-sdk/`. Adding a new event type means updating the union here AND the server emitter together.
- `run-reducer.ts` is **pure** — events in, state out. Unit-test it. Do not put side effects in it.
- The component subscribing to a run holds a `useReducer` of `Run` state, mounts an `AbortController`, calls `consumeRun` in an effect, and aborts on unmount.
- On reconnect, call `consumeRun` again with the last seen event id. The server replays missed events.

### Cross-cutting UI state

- Use React Context only for genuinely cross-cutting *UI* state (theme, command palette, sidebar collapse).
- Server data is not UI state. It belongs in TanStack Query.
- Single-screen state stays in `useState` / `useReducer`. Don't lift to Context just because two siblings need it — pass props or compose components.

### Styling

- Utility-first Tailwind. No `@apply` for component reuse — extract a React component instead.
- Design tokens in `tailwind.config.ts`; semantic colors as CSS custom properties so dark mode is one variable swap.
- Build small primitives under `components/ui/`. Don't pull in shadcn/MUI without explicit user approval.

### Forms

- Controlled inputs in small forms. `useReducer` for multi-field forms.
- Validation lives on the server. Reflect server errors back into the form.
- Add `react-hook-form` only when a form has 5+ fields with cross-field validation. Get user approval before adding the dep.

### Errors

- Server errors arrive as `ApiError` with `{ status, code, message, details }`. Surface them — never swallow.
- Read errors via TanStack Query's `error` field in components.
- An SSE `error` event ends the run; the reducer transitions to `error` state. Don't auto-retry stream errors silently.

### Testing

- Co-locate tests next to the file they cover (`Foo.tsx` ↔ `Foo.test.tsx`). No parallel `__tests__/` tree.
- Pure helpers (`sse.ts`, `run-reducer.ts`, query-key factory) get full coverage — they're load-bearing.
- Components: query by accessible role/label, not by class or test-id where avoidable. Mock at the API boundary (stub `apiFetch`), not at the component level.
- No Playwright by default. Get user approval before adding it.

### Imports

- Use `@/...` for anything outside the current directory's siblings. Use relative paths for siblings only.
- No second alias. One is enough.
